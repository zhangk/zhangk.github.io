## Settings Provider 简介

Settings Provider 是 Android 系统中用于存储和共享系统设置的数据库，通过以下 API 进行操作：

```java
// 整型设置
Settings.System.putInt(getActivity().getContentResolver(), "key", 1);

// 字符串设置
Settings.System.putString(getActivity().getContentResolver(), "key", "value");

// 长整型设置
Settings.System.putLong(getActivity().getContentResolver(), "key", 1L);

// 浮点型设置
Settings.System.putFloat(getActivity().getContentResolver(), "key", 1.0f);
```

## Settings 的三种分类

| 类型         | 描述        | 权限         |
| ---------- | --------- | ---------- |
| **Global** | 系统全局设置    | 第三方应用可读不可写 |
| **System** | 用户偏好系统设置  | 应用可读写自己的设置 |
| **Secure** | 安全性用户偏好设置 | 第三方应用可读不可写 |

## 查看 Settings Provider 值的方法

### 1. 通过代码查看

```
// 获取系统设置
int timeout = Settings.System.getInt(
    getContentResolver(), 
    Settings.System.SCREEN_OFF_TIMEOUT
);
```

### 2. 直接访问数据库文件

**Android 5.x 平台**:

```
adb shell
su
cd /data/data/com.android.providers.settings/databases
ls -l settings.db
```

**Android 7.x+ 平台**:

```
adb shell
su
cd /data/system/users/0
ls -l settings_*.xml
```

### 3. 使用 ADB 命令（推荐）

```bash
# 基本语法
adb shell settings [--user <USER_ID> | current] <command> <namespace> <key> [value]
```

## ADB 操作 Settings 命令详解

### 查看设置值

```bash
# 查看系统设置
adb shell settings get system screen_off_timeout

# 查看全局设置
adb shell settings get global airplane_mode_on

# 查看安全设置
adb shell settings get secure bluetooth_address
```

### 修改设置值

```bash
# 修改系统设置（屏幕超时时间）
adb shell settings put system screen_off_timeout 120000

# 修改全局设置（飞行模式）
adb shell settings put global airplane_mode_on 1

# 应用更改（需要广播通知系统）
adb shell am broadcast -a android.intent.action.AIRPLANE_MODE
```

### 删除设置值

```bash
# 删除特定设置
adb shell settings delete system screen_off_timeout

# 删除后会恢复默认值
```

### 列出所有设置

```bash
# 列出所有系统设置
adb shell settings list system

# 列出所有全局设置
adb shell settings list global

# 列出所有安全设置
adb shell settings list secure
```

## 实用示例

### 1. 屏幕超时时间管理

```bash
# 查看当前屏幕超时时间（毫秒）
adb shell settings get system screen_off_timeout

# 设置屏幕超时为2分钟（120,000毫秒）
adb shell settings put system screen_off_timeout 120000

# 恢复默认设置
adb shell settings delete system screen_off_timeout
```

### 2. 飞行模式控制

```bash
# 开启飞行模式
adb shell settings put global airplane_mode_on 1
adb shell am broadcast -a android.intent.action.AIRPLANE_MODE

# 关闭飞行模式
adb shell settings put global airplane_mode_on 0
adb shell am broadcast -a android.intent.action.AIRPLANE_MODE
```

### 3. 查看设备信息

```bash
# 获取设备ID
adb shell settings get secure android_id

# 获取蓝牙地址
adb shell settings get secure bluetooth_address

# 获取默认输入法
adb shell settings get secure default_input_method
```

## 常见问题解决

### 1. 返回 `null` 值

```bash
adb shell settings get system invalid_key
# 返回：null
```

**原因**：键名不存在或拼写错误

### 2. 权限不足

```bash
adb shell settings put secure android_id "new_id"
# 错误：Permission denial
```

**原因**：Secure 设置需要系统权限才能修改

### 3. 多用户支持

```bash
# 指定用户ID操作
adb shell settings --user 10 get system screen_off_timeout

# 操作用户0（系统用户）
adb shell settings --user 0 put system screen_off_timeout 300000
```

## 高级技巧

### 1. 批量导出设置

```bash
# 导出所有系统设置到文件
adb shell settings list system > system_settings.txt

# 导出所有全局设置
adb shell settings list global > global_settings.txt
```

### 2. 对比设置差异

```bash
# 导出设备A的设置
adb -s deviceA shell settings list system > deviceA_system.txt

# 导出设备B的设置
adb -s deviceB shell settings list system > deviceB_system.txt

# 对比差异
diff deviceA_system.txt deviceB_system.txt
```

### 3. 重置所有设置

```bash
# 重置所有设置（谨慎使用！）
adb shell settings reset system
adb shell settings reset global
adb shell settings reset secure
```

## 注意事项

1. **系统权限**：修改 Secure 和 Global 设置需要 root 权限

2. **数据类型**：确保使用正确的数据类型（int/string/long/float）

3. **用户影响**：设置变更会立即生效，影响所有用户

4. **系统稳定性**：错误设置可能导致系统不稳定

5. **备份恢复**：重要修改前建议备份原始设置

通过掌握这些 ADB 操作 Settings 的技巧，可以高效调试系统设置问题，验证 ROM 开发中的修改效果。
