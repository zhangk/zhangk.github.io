## 📝 问题描述

安兔兔等检测软件显示电池容量不正确，需修改系统电源配置文件中的电池容量值。

## 📂 修改文件

`frameworks/base/core/res/res/xml/power_profile.xml`

## 🔧 修改内容

```diff
--- a/core/res/res/xml/power_profile.xml
+++ b/core/res/res/xml/power_profile.xml
@@ -80,7 +80,7 @@
         <value>4</value> <!-- Cluster 0 has 4 cores (cpu0, cpu1, cpu2, cpu3) -->
     </array>
     <!-- This is the battery capacity in mAh -->
-    <item name="battery.capacity">9000</item>
+    <item name="battery.capacity">3500</item>
 
     <array name="wifi.batchedscan">
         <value>.0003</value> <!-- 1-8/hr -->
```

## ⚙️ 关键参数说明

```mermaid
graph LR
    A[安兔兔读取] --> B[power_profile.xml]
    B --> C[battery.capacity]
    C --> D[显示电池容量]
    
    style C fill:#f96,stroke:#333,stroke-width:2px
```

| 参数                   | 原值   | 新值   | 单位  | 作用        |
| :------------------- | :--- | :--- | :-- | :-------- |
| **battery.capacity** | 9000 | 3500 | mAh | 系统报告的电池容量 |

## ⚠️ 注意事项

1. **实际容量匹配**：\
   确保 `3500` 与实际电池物理容量一致

2. **单位要求**：\
   值必须以 **mAh** (毫安时) 为单位

3. **覆盖层机制**：

   - 修改overlay目录下的文件而非原始文件

   - 系统优先使用overlay中的值

4. **其他影响**：\
   此修改还会影响：

   - 系统设置中的电池信息

   - 第三方电池检测工具

   - 系统电量统计精度

5. **编译要求**：

   bash

   ```
   make clean && make -j8  # 需要重新编译系统
   ```

> 🔍 **验证方法**：\
> 修改后可通过以下命令检查：
>
> ```bash
> adb shell grep "battery.capacity" /system/framework/framework-res.apk
> ```
>
> 应显示：`<item name="battery.capacity">3500</item>`
