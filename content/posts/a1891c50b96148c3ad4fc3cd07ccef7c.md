## 一、设置默认时区

### 核心配置

在 `device.mk` 或 `build/target/product/` 相关文件中添加：

```makefile
PRODUCT_PROPERTY_OVERRIDES += \
    persist.sys.timezone=Asia/Shanghai
```

## 二、设置默认语言

### 核心配置

```makefile
PRODUCT_PROPERTY_OVERRIDES += \
    ro.product.locale.language=EN \
    persist.sys.language=EN \
    persist.sys.country=US \
    ro.product.region=US
```

## 三、验证配置

### 检查系统属性

```bash
adb shell getprop | grep -E "timezone|language|country|region"

# 预期输出示例：
#[persist.sys.timezone]: [Asia/Shanghai]
#[persist.sys.language]: [en]
#[persist.sys.country]: [US]
#[ro.product.locale.language]: [EN]
```

## ⚠️ 注意事项

1. **大小写敏感**：

   - 时区值：`Asia/Shanghai`（正确） vs `asia/shanghai`（错误）

   - 语言代码：`zh-CN`（正确） vs `zh-cn`（错误）

2. **属性优先级**：

   ```mermaid
   graph LR
   A[ro.product.locale] -->|最高| B[系统设置]
   C[persist.sys.language] --> D[build.prop]
   E[默认区域设置] -->|最低| F[系统资源]
   ```

3. **首次启动逻辑**：

   - `persist.sys.*` 属性在首次启动后生效

   - 恢复出厂设置会重置为build.prop中的值

4. **兼容性测试**：

   - 时区切换后验证自动时间更新

   - 语言切换后检查RTL布局（阿拉伯语等）
