## 操作前提

1. 已进入Android源码编译环境（`source build/envsetup.sh` && `lunch`）

2. 已连接设备，`adb`可用

## 1. 单编Services模块

在源码根目录执行：

```或指定编译路径：
make services
```

或指定编译路径：

```bash
make services -j\$(nproc) # -j后接线程数，加速编译
```

**编译输出路径：**\
`out/target/product/<product_name>/system/framework/services.jar`

## 2. 推送更新到设备

1. 重新挂载`/system`分区为可写（**需root或已解锁system分区**）：

```bash
adb root  
adb remount  
# 或 adb disable-verity 后重启
```

推送编译生成的`services.jar`：

```bash
adb push out/target/product/product_name/system/framework/services.jar /system/framework/
```

1. *注意：将`<product_name>`替换为实际的产品名（如`generic_x86_64`、`sdk_phone_x86_64`等）*

## 3. 重启相关进程使修改生效

### 方式一：重启系统服务（推荐）

```bash
adb shell stop  
adb shell start
```

### 方式二：重启整个系统

```bash
adb reboot
```

### 方式三：仅重启受影响的进程（不保证完全生效）

```bash
adb shell killall system_server
```
