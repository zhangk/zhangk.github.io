## 修改目标

将系统默认时间格式从12小时制改为**24小时制**

## 修改文件及路径

### 1. 修改默认值配置文件

**路径**: `frameworks/base/packages/SettingsProvider/res/values/defaults.xml`\
**修改内容**:

```xml
@@ -223,4 +223,5 @@
 
     <!--Default for Settings.System.BUTTON_LIGHTS_OFF_TIMEOUT-->
     <integer name="def_button_lights_off_timeout">1500</integer>
+    <string name="time_12_24">24</string>
 </resources>
```

**关键说明**：

- 添加`<string name="time_12_24">24</string>`定义默认值

- 值"24"表示使用24小时格式（若为"12"则表示12小时制）

### 2. 修改数据库初始化代码

**路径**: `frameworks/base/packages/SettingsProvider/src/com/android/providers/settings/DatabaseHelper.java`\
**修改内容**:

```java
@@ -2359,6 +2359,7 @@ public class DatabaseHelper extends SQLiteOpenHelper {
 
             loadIntegerSetting(stmt, Settings.System.POINTER_SPEED,
                     R.integer.def_pointer_speed);
+            loadStringSetting(stmt, Settings.System.TIME_12_24, R.string.time_12_24);
         } finally {
             if (stmt != null) stmt.close();
         }
```

**关键说明**：

- 添加`loadStringSetting()`调用加载时间格式设置

- 使用`Settings.System.TIME_12_24`系统属性键

- 关联到前面定义的`R.string.time_12_24`资源

## 生效验证

1. 编译刷机后进入设置：

   ```sh
   adb shell settings get system time_12_24  # 应返回"24"
   ```

2. 检查系统UI：

   - 状态栏时间显示应为24小时格式（如14:30而非2:30 PM）

   - 设置→日期和时间→时间格式应为24小时制

## 注意事项

1. 需要**整机编译**才能生效：

   ```sh
   make -j8
   ```

2. 如果设备已存在用户设置（如曾经手动修改过时间格式），此默认值不会覆盖用户设置

3. 完整属性路径：`Settings.System.TIME_12_24`
