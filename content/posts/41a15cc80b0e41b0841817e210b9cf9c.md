## ❌ 常见错误做法

```git
git branch -r -d origin/branch-name
```

> **效果**：仅删除本地对远程分支的跟踪引用，**不会实际删除远程分支**

## ✅ 正确删除方法

### 方法1：空分支推送法 (通用)

```git
git push origin :branch-name
```

> **注意**：冒号前必须有空格，原理是用空内容覆盖远程分支

### 方法2：--delete 参数法 (推荐)

```git
git push origin --delete branch-name
```

### 方法3：完整URL格式

```git
git push https://github.com/user/repo.git --delete branch-name
```

## 🔍 操作原理图解

```text
本地仓库          ---(空分支推送)-->       远程仓库
(无分支内容)                           (删除目标分支)
```

## 🛠️ 完整操作流程

1. 查看远程分支：

   ```git
   git branch -r
   ```

2. 执行删除操作：

   ```git
   git push origin --delete feature/temp
   ```

3. 验证删除结果：

   ```git
   git fetch --prune  # 更新远程分支引用
   git branch -r      # 确认分支已消失
   ```

## ⚠️ 重要注意事项

1. **权限要求**：必须拥有远程仓库的推送权限

2. **保护分支**：无法删除被设为`Protected Branch`的分支

3. **同步本地**：删除远程分支后建议清理本地跟踪：

   ```git
   git fetch --prune
   ```

4. **恢复方法**：误删后可通过commit ID恢复：

   ```git
   git push origin <commit-hash>:refs/heads/branch-name
   ```

## 💡 实用技巧

- **批量删除**：使用通配符删除多个分支

  ```git
  git push origin --delete feature/*
  ```

- **查看操作记录**：确认删除结果

  ```git
  git ls-remote origin
  ```

- **GUI工具操作**：

  - VSCode：分支视图 → 右键删除

  - GitKraken：右键分支 → Delete

> **Git版本差异**：\
> Git 1.7.0+ 支持`--delete`语法\
> 旧版本必须使用空分支推送法
