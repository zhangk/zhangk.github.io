## 修改说明

1. **原始行为**：

   - 使用 `ANDROID_ID` 前4位字符生成动态名称

   - 格式：`Android_XXXX`（例如 `Android_A3F2`）
2. **修改后行为**：

   - 使用固定名称：**"Archos Access 101 Wifi"**

   - 所有设备将显示相同的 WiFi 直连名称

## 影响范围

| 功能             | 是否受影响  |
| -------------- | ------ |
| WiFi Direct 名称 | ✅ 直接修改 |
| 普通 WiFi 热点名称   | ❌ 不影响  |
| 蓝牙设备名称         | ❌ 不影响  |
| 设备型号显示         | ❌ 不影响  |

## 编译部署步骤

1. 修改源码文件：

   bash

   ```
   vim frameworks/opt/net/wifi/service/java/com/android/server/wifi/p2p/WifiP2pServiceImpl.java
   ```

2. 重新编译 WiFi 服务模块：

   ```bash
   mmm frameworks/opt/net/wifi/
   ```

3. 推送更新到设备：

   ```bash
   adb push out/target/product/<device>/system/framework/services.jar /system/framework/
   adb reboot
   ```

## 验证方法

1. 开启 WiFi Direct：

   ```java
   WifiP2pManager manager = (WifiP2pManager) getSystemService(WIFI_P2P_SERVICE);
   manager.discoverPeers(channel, null);
   ```

2. 检查设备名称：

   ```bash
   adb shell dumpsys wifi | grep "Device Name"
   # 预期输出：Device Name: Archos Access 101 Wifi
   ```

## 注意事项

1. **名称冲突**：所有修改后的设备将显示相同名称，可能造成识别困难

2. **合规性**：部分国家/地区要求 WiFi 名称需包含设备唯一标识符

3. **恢复默认**：如需恢复原始行为，撤销修改即可

4. **替代方案**：可通过系统属性动态配置：

   ```java
   return SystemProperties.get("persist.wifi.direct.name", "Archos Access 101 Wifi");
   ```
